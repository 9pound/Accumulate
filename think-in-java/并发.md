# 并发

顺序编程：程序中的所有事物在任意时刻都只能执行一个步骤

并发：要么看起来在并发执行，要么在多处理器环境下可以同时执行



为什么要用并发

- 速度
- 设计可管理性



并发通常是提高运行在单处理器上程序的性能

阻塞

- 如果程序中的某个任务因为该程序控制范围之外的某些条件而导致不能继续执行，那么我们就说这个任务或线程阻塞了。
- 如果没有任务会阻塞，那么在单处理器上使用并发就没有意义。
