# 对象的共享



可见性：为了确保多个线程之间对内存写入操作的可见性，必须使用同步机制



失效数据：

get/set 方法都需要同步

最低安全性：当线程在没有同步的情况下读取变量时，可能会得到一个失效值，但至少这个值是由之前某个线程设置的值，而不是一个随机值



非原子64位操作：

java内存模型要求，变量的读取操作和写入操作都必须是原子的

但是对于非volatile类型的long和double变量，jvm允许将64位的读操作或写操作分解为两个32位的操作

#### Volatile变量

弱同步机制，用来确保将变量的更新操作通知到其他线程

不建议过度依赖volatile变量提供的可见性

何时使用volatile变量

- 对变量的写入操作不依赖变量的当前值，或者你能确保只有单个线程更新变量的值
- 该变量不会与其他状态变量一起纳入不变性条件中
- 在访问变量时不需要加锁



### 发布与逸出

发布：使对象能够在当前作用域之外的代码中使用 

逸出：当某个不应该被发布的对象被发布的时候



发布的三种形式： （this引用逸出）
