# 基础

1、线程安全



线程安全的核心在于，要对状态（共享的或者可变的状态）的访问操作进行管理

对象的状态，是指存储在状态变量中的数据

共享：变量可以被多个线程访问

可变：变量的值在生命周期内可以发生变化

编写并发应用程序时，正确的编程方法，首先使代码正确运行，然后再提高代码的速度，即便如此，最好也只是当性能测试结果，应用需求告诉你必须提高性能，以及测量结果表明这种优化在实际环境中确实能带来性能提升时，才进行优化。

线程安全程序





单线程的正确性：所见即所知，we know it when we see it

线程安全的定义：当多线程访问某个类时，这个类始终都能表现出正确的行为，那么就称这个类是线程安全的



结论：无状态对象一定是线程安全的



不恰当的执行而出现不正确的结果是一种非常重要的情况，他有一个正式的名字，竞态条件，正确的结果取决于运气

竞态条件的类型 

- 先检查后执行，通过一个可能失效的观测结果来决定下一步的动作。
- 读取-修改-写入，基于对象之前的状态来定义对象状态的转换

竞态条件的本质：基于一种可能失效的观察结果来做出判断或者执行某个计算，这种类型的静态条件称为先检查后执行
