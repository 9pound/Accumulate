## 线程间的通信机制

- 共享内存（隐式同步）
  - 线程之间通过读-写内存中的公共状态来隐式通信。volatile共享内存

- 消息传递（显示同步）
  - 线程之间必须通过明确的发送信息来显示的进行通信。wait/notify等待通知方式

- 管道流：管道输入/输出流的形式。

## 重排序

### 定义

是指编译器和处理器为了优化程序性能而对指令序列进行重新排序的一种手段

### 分类

- 编译器重排序
  - 编译器优化重排序
- 处理器重排序
  - 指令级并行重排序
  - 内存系统重排序

JMM的编译器重排序规则会禁止特定类型的编译器重排序

java编译器在生成指令序列时，插入指定类型的内存屏障指令来禁止特定类型的处理器重排序，



### 为什么会重排序？

处理器使用写缓冲临时保存向内存写入的数据，但是写缓冲仅对自己的处理器可见，它会导致处理器执行内存操作的顺序可能会与内存实际操作的执行顺序不一致。所以处理器允许对读-写操作进行重排序。

### 处理器重排序规则

- load-load
- load-store
- store-store
- store-load

## 先行发生原则

JSR-133使用happens-before的原则用来描述操作之间的可见性，如果一个操作的执行结果需要对另一个操作可见，那么这两个操作之间必须要存在happen-before关系

- 程序顺序规则
- 监视器锁规则
- volatile变量规则
- 传递性



## 顺序一致性

如果程序时正确同步的，程序的执行将具有顺序一致性。即程序的执行结果与该程序在顺序一致性内存模型中的执行结果相同

### 顺序一致性内存模型

- 一个线程的所有操作必须按照程序的顺序来执行
- 不管程序是否有同步，所有的线程都只能看到一个单一的操作执行顺序，每个操作都必须原子执行，切立刻对所有线程可见







as-if-serial与happens-before的区别
下面来比较一下as-if-serial和happens-before：

as-if-serial VS happens-before

as-if-serial语义保证单线程内程序的执行结果不被改变，happens-before关系保证正确同步的多线程程序的执行结果不被改变。
as-if-serial语义给编写单线程程序的程序员创造了一个幻境：单线程程序是按程序的顺序来执行的。happens-before关系给编写正确同步的多线程程序的程序员创造了一个幻境：正确同步的多线程程序是按happens-before指定的顺序来执行的。
as-if-serial语义和happens-before这么做的目的，都是为了在不改变程序执行结果的前提下，尽可能地提高程序执行的并行度。
