# Seata

一款开源的分布式事务解决方案，为开发者提供一栈式的分布式事务解决方案，提供了AT，TCC，Saga，XA事务模式



## AT模式：

定义：基于XA演进而来的一种分布式事务模式

原理：

TM：事务管理器

负责向TC注册全局事务，并生成一个全局唯一的XID

RM: 

在业务层面通过JDBC标准接口访问RM时，SeaTa会对所有请求进行拦截，每个本地事务提交时TM都会向TC注册一个分支事务

TC: TC就是SeaTa服务，独立部署

执行流程：

1. TM向TC注册全局事务，并生成全局唯一ID。
2. RM向TC注册分支事务，并将其纳入该XID对应的全局事务范围。
3. RM向TC汇报资源准备状态。
4. TC汇总所有事务参与者的执行状态，决定分布式事务是全部回滚还是提交
5. TC通知所有RM提交或者回滚
