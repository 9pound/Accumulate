# 线程

本质：线程是进程中的一个控制流，控制流就是程序计数器指向的位置

为什么要有线程？

- 利用处理器多核心，提高速度
- 解决阻塞问题，提高系统响应性

## 性质

- 操作系统调度的最小单元



### 线程的优先级

- 频繁阻塞（休眠或者IO操作）的线程需要设置较高优先级，偏重计算的线程设置较低优先级，确保处理器不会被独占

### 线程的状态



六种



### Daemon线程

用于后台调度

- Daemon属性需要在启动线程之前设置，不能在启动线程之后设置
- Daemon线程不能依靠finally块中的内容来确保执行关闭或者清理资源的逻辑（Java虚拟机退出时daemon线程中的finally块不一定会被执行）

### 中断线程

中断的本质：就是线程的一个标志位属性，其他线程通过调用该线程的interrupt()方法对其进行中断操作



isInterrupted() :判断是否被中断



Thread.interrupted()：对当前线程中断标志位进行复位
